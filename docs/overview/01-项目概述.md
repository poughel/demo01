# DataSophon 项目概述

## 一、项目简介

DataSophon 是一个用于快速部署、管理、监控和自动化运维大数据服务组件和节点的平台，帮助用户快速构建稳定、高效的大数据集群服务。

### 项目名称来源
项目名称灵感来源于刘慈欣的科幻小说《三体》中的"智子"（Sophon）。在小说中，智子是三体文明部署在地球的实时远程监控管理平台。DataSophon 类似地提供了对大数据基础设施组件和节点的自动化监控、运维管理功能。

## 二、核心特性

1. **易于部署**: 可快速完成约300个节点的大数据集群部署
2. **国产化兼容**: 兼容ARM服务器和常见国产化操作系统
3. **全面监控**: 丰富的监控指标，基于生产实践
4. **灵活告警**: 支持自定义告警组和告警指标
5. **强扩展性**: 用户可通过配置集成或升级任何组件

## 三、技术架构

### 整体架构
DataSophon 采用分层架构设计，主要包括以下层次：
- **API 层**: 提供 RESTful API 接口
- **服务层**: 核心业务逻辑处理
- **领域层**: 领域模型定义
- **基础设施层**: 数据持久化、缓存等
- **Worker 层**: 分布式工作节点

### 技术栈
- **后端框架**: Spring Boot
- **前端框架**: Vue.js
- **构建工具**: Maven
- **数据库**: MySQL
- **消息队列**: (待分析)
- **监控**: Prometheus + Grafana

## 四、项目模块结构

```
datasophon/
├── datasophon-api/           # API服务模块，提供HTTP接口
├── datasophon-common/         # 公共组件模块，工具类和通用功能
├── datasophon-domain/         # 领域模型模块，实体定义
├── datasophon-infrastructure/ # 基础设施模块，数据访问层
├── datasophon-service/        # 业务服务模块，核心业务逻辑
├── datasophon-worker/         # 工作节点模块，执行具体任务
├── datasophon-init/           # 初始化模块，部署和初始化脚本
├── datasophon-ui/             # 前端UI模块，Web界面
├── bin/                       # 可执行脚本
├── conf/                      # 配置文件
├── deploy/                    # 部署相关文件
├── docs/                      # 文档
└── scripts/                   # 辅助脚本
```

## 五、核心功能模块

### 5.1 集群管理
- 集群创建、配置、启动、停止
- 节点管理和监控
- 服务组件管理

### 5.2 服务组件支持
支持的大数据组件包括但不限于：
- **存储**: HDFS, HBase
- **计算**: YARN, Spark, Flink
- **数据仓库**: Hive, Doris, Iceberg
- **消息队列**: Kafka
- **协调服务**: ZooKeeper
- **搜索引擎**: Elasticsearch
- **监控告警**: Prometheus, Grafana, AlertManager
- **数据集成**: DataX, SeaTunnel
- **查询引擎**: Trino, Kyuubi
- **安全**: Kerberos, Ranger

### 5.3 监控告警
- 实时监控服务状态
- 自定义监控指标
- 灵活的告警规则配置
- 告警组管理

### 5.4 运维管理
- 服务启停控制
- 配置管理
- 日志查看
- 性能优化

## 六、部署架构

DataSophon 采用 Master-Worker 架构：
- **Master 节点**: 运行 datasophon-api 服务，提供管理界面和API
- **Worker 节点**: 运行 datasophon-worker 服务，执行具体的部署和管理任务

## 七、开发语言和工具

- **主要开发语言**: Java 8+
- **前端语言**: JavaScript (Vue.js)
- **构建工具**: Maven 3.x
- **版本控制**: Git
- **代码风格**: 遵循阿里巴巴Java开发规范
